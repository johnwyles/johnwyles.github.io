Setting up Kubernetes and OpenFaas on a Raspberry Pi cluster using Raspian

Mar 27, 2019
2 min read

NOTE: This is a work in progress and this guide is not complete

NOTE: This guide was based on a gist by GitHub user alexellis for setting up Kubernetes on Raspbian as well as a gist getting started with Raspian for each node which I have made some minor additions to in this post.

Assumptions

  • You have more than 1 Raspberry Pi with at least an 8GB SD card for each
  • You want all of your Raspberry Pi’s to have a static IP
  • You desire all of your networking and orchestration over WIFI rather than Ethernet
  • For troubleshooting that you have access to your routers administrative panel to see attached devices and their IP address
  • For further troubleshooting you may require Ethernet access for diagnosis and further setup

Steps

  1. Clone my Raspberry Pi Kubernetes Cluster repository and change to the raspian directory: 
    1
2
3

        git clone https://github.com/johnwyles/raspberry-pi-kubernetes-cluster.git
    cd raspberry-pi-kubernetes-cluster/raspbian
  2. Download the latest Raspbian Lite image (as of writing this was: 2018-11-13 Raspbian Stretch Lite ) to the raspberry-pi-kubernetes-cluster/raspbian directory we cd‘ed into above: Raspbian Images
  3. Insert an SD card you are ready to flash to and flash the SD card with the image:
    • If you are on a Mac use a software like balena Etcher to flash the image to only one of the Raspberry Pi SD cards that will be in your cluster. This machine will allow us to image all of the subsequent Raspberry Pi machines in this cluster.
    • If you are on a Linux machine use dd: 
      1
2
3
4
5

          # Change "2018-11-13-raspbian-stretch-lite.img" to whatever the latest
    # image file name is and "/dev/sda" to the device output from the `lsblk`
    # command when
    dd if=2018-11-13-raspbian-stretch-lite.img of=/dev/sda bs=1M
  4. Remount the SD card now that it has been imaged:
    • On Mac simply unplugging and replugging in the card should do
    • On Linux (substituting /dev/sda1 for whatever device shows up in the output from lsblk): 
      1
2
3
4
5

            umount /dev/sda1
      umount /dev/sda2
      mkdir -p /mnt/boot
      mount /dev/sda1 /mnt/boot
  5. Eject or unmount the SD card and put it into a Raspberry Pi connected by Ethernet and power it on
  6. SSH to the machine using (login: _pi) password: raspberry): 
    1
2

        ssh -l pi 192.168.1.101
